01 ゴール / 絶対に避けたい3つの事故

「これが起きたら失敗」を先に決める

機能を足す前に、何を絶対に起こしてはいけないかを宣言する。以降のすべての要件は、この3つのゴールを守るために存在する。

GOAL 1

二重課金しない

ボタン連打や通信エラーの再送でも、お客様のカードに2回請求しない。

GOAL 2

売り越さない

在庫1個に2人が同時注文しても、確定するのは1人だけ 在庫引当。

GOAL 3

注文を取りこぼさない

決済が成功したのに注文が記録されない、を起こさない。

02 範囲 / だれが関わるか

この文書が扱う範囲と、登場人物

09 用語集 / この文書で使う専門語

本文に出てくる専門語のやさしい説明

本文中の専門語は、ここに必ず登録されている (用語の唯一の出どころ)。

在庫引当stock allocation
注文の瞬間に在庫を「この人の分」として押さえること。押さえないと同じ1個を2人に売ってしまう。
二重課金double charge
同じ買い物で2回お金を請求してしまう事故。連打や通信再送で起こりやすい。
WMSwarehouse mgmt
倉庫の在庫を管理する既存システム。在庫数の正しい出どころ。
PCI DSScard security std
カード情報を扱うときに守る世界共通の安全ルール。違反すると決済できなくなる。
