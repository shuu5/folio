architecture/research/ — 要望 + 調査資料

v0.4.2-draft · user 要望 + 業界調査 + plugin scope の集約

本 dir は試作 plugin 開発のための 情報集約場所。 user 要望、 業界ベストプラクティス調査、 plugin scope 候補等を HTML 形式で随時追加する。 試作 plugin が完成したら撤去 or archive。

§1. Purpose

試作 plugin の開発に必要な情報を user-Claude 対話で集める場所。 議論を memo 化、 業界調査結果を整理、 plugin の最小 scope を確定する材料を蓄積する。

§2. Contents (随時追加)

予定 (試作の進行に応じて変動):

  • user-requirements.html — user が folio に求める機能リスト (対話で集約)
  • industry-best-practices.html — OpenSpec / Kiro / Spec Kit / ESLint 等の業界調査
  • plugin-minimum-scope.html — plugin の最小機能候補
  • (その他、 試作中に必要に応じて追加)

命名は kebab-case.html 推奨 (exploration 領域なので柔軟、 ただし reader 体験のため統一)。

§3. Rules

  • exploration 領域 (research/) なので folio rule に縛られない。 markup 正規性、 declarative form 等は不要、 自由に書く。
  • file 内容は random draft / 思考 memo / 引用 / link 等 何でも可。
  • 基本 HTML で書く (Markdown は research/ 内では避ける、 visual scan 統一のため)。
