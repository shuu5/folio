architecture — index

folio は、 人間と AI agent が同じ architecture spec を読み書きするための spec-writing framework です。 仕様は HTML + JSON-LD で記述され、 機械検証 (folio validate) と人間可読性 (P-14) の両方を恒久ゲートで保ちます。

まずここから — 導入手順 (repo README)

初めて folio を使う →導入手順と全体像 (repo README) 仕様・要件を調べる →spec — rules / relations / verification (WHAT) 設計の経緯を追う →decisions — Architecture Decision Records (WHY)

Key documents

  • constitutionfolio の 14 不変原則。 すべての設計判断の最上位根拠 (編集禁止)
  • rulesspec の書き方の規範 — markup・命名・EARS 要件・検証の universal rules
  • folio-self-specfolio 自身の architecture spec (framework の self-host 実例)
  • verification仕様がどう機械検証されるか — validate / sandbox framework の contract
  • glossaryfolio 用語集 — 本文の用語 hover 解説と同じ定義から生成
