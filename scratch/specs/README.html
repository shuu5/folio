specs/ — folio framework spec drafts

v0.4.2-draft · ブラッシュアップ対象の folio framework spec を集約

本 dir は folio framework 自身の spec (constitution 以外) を draft 状態で集約する。 試作段階 (Phase X2) 中はここで自由にブラッシュアップする。 完成後 (Phase X3) に正式 location (architecture/) に移植予定。

§1. Purpose

constitution が 不変資産 として固定された一方、 rules / folio-self-spec は試作の進行に応じて 継続的にブラッシュアップする必要がある。 両者を物理的に分離 (constitution = scratch/ 直下 / specs = scratch/specs/) することで、 「触らない」 と 「改訂可」 の境界を明示する。

§2. Contents

filerolestatus
rules.html Layer 1 consumer project 向け universal rules (markup / naming / EARS / Mandatory Actions) draft
folio-self-spec.html folio Layer 0 framework 自身の architecture spec (provisional) draft

§3. Rules

  • 編集可: 試作の進行に応じて自由にブラッシュアップする。 markup 正規性、 SemVer bump 等の formal 要件は scratch/ 段階では緩い適用。
  • 大幅改訂は ADR (../decisions/) で記録するが、 軽微な refinement は inline 修正で OK。
  • constitution との関係: rules / folio-self-spec は constitution の不変原則 (P-1〜P-12) を 実装する 位置付け。 不整合が出たら user と相談。

§4. Lifecycle

  • 現在 (Phase X2): 試作と並行してブラッシュアップ中。
  • 完成後 (Phase X3): architecture/spec/ 配下に self-application で分解、 正式化。 scratch/specs/ は撤去 or archive。
