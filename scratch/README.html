scratch/ — 試作・工事用箱

v0.4.2-draft · Phase X3 試作段階の作業場所

本 dir は folio framework が試作段階にある期間の 工事用一時的箱。 試作・調査・要望整理・実装試行は すべてここで行う。 完成後 (Phase X4+) に正式 location (architecture/) に移植、 scratch/ は撤去予定。

§1. Purpose

folio framework は self-application paradox (自身を縛る rule で自身を書けない) を抱えている。 試作段階では folio rule に縛られず自由に書ける場所として scratch/ を採用する。

§2. Contents

scratch/ 直下:

  • constitution.html — 12 不変原則 (編集禁止、 特別枠)
  • x4-plan.html — Phase X4 分解 + 着手計画 (planning memo、 X4 着手時の起点)
  • assets/mermaid.min.js — vendor

サブ dir:

  • specs/ — folio framework spec drafts (rules.html / folio-self-spec.html / relations.html / verification.html + README.html、 ブラッシュアップ可)
  • decisions/ — ADR cluster (frozen、 新規起票は user 承認 MUST)
  • research/ — user 要望 + 業界調査 (HTML 形式、 自由形式)
  • verification/ — Phase X3 sandbox verification framework (scenarios/ / fixtures/ / baselines/ + runner.sh、 ADR-0013)

試作 plugin 本体は scratch/ ではなく plugin root 直下 (.claude-plugin/ + hooks/) で実装 (Phase X3、 ADR-0003)。 完成形 (Phase X4+) で architecture/ へ移植予定。

§3. Rules

  • scratch/ 内は folio rule に縛られない。 markup 正規性、 命名規約、 declarative form 等の rule は適用しない。 自由に書く。
  • 例外: constitution.html + decisions/ 配下既存 ADR は 編集禁止 (frozen 資産)。
  • specs/ 配下 (rules.html / folio-self-spec.html) は ブラッシュアップ可。 試作の進行に応じて自由に改訂する。
  • 新規 file / dir 作成は自由。 思考 memo / 一時 work は柔軟に。

§4. Lifecycle

  • 現在 (Phase X3): 試作・調査の作業場所として active。 試作 plugin 本体は .claude-plugin/ + hooks/ で実装中。
  • 完成後 (Phase X4+): 試作 plugin の残部を plugin root 直下へ正式整理、 folio 自身の architecture spec を architecture/ に移植。 scratch/ は撤去 or archive。
