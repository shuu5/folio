decisions/ — Architecture Decision Records

v0.4.2-draft · folio Layer 0 の ADR cluster

本 cluster は folio framework に関する architectural decision を記録する Architecture Decision Records (ADR) を保持する。 rules.html §3 File Naming Convention に従い、 各 ADR は ADR-NNNN-<slug>.html の命名で配置する。

§1. Purpose

decisions/ は constitution P-7 (Content domain exclusivity) における「decisions = why」 領域を担う:

  • spec (constitution / rules / folio-self-spec) = what (構造・不変条件)
  • decisions (本 cluster) = why (frozen ADR、 architectural decisions)
  • changes (将来 cluster、 未作成) = proposal (進行中の更新提案)
  • research (将来 cluster、 未作成) = exploration (spec 反映前の探索)

§2. ADR Status Lifecycle

status意味
proposeddraft 段階、 user 承認待ち
accepteduser 承認済、 適用済
deprecated古い ADR、 後続 ADR で superseded された
superseded明示的に新 ADR で置換された (supersedes 属性で trace)

§4. Naming Convention

ADR file 名は ADR-NNNN-<slug>.html 形式 (4 桁連番 + kebab-case slug)。 rules.html §3 File Naming Convention を参照。

§5. ADR Creation Procedure

constitution §6 Amendment Procedure に従う:

  1. 変更提案を changes/ 配下に proposal として起案 (proposal-only MUST)。
  2. user 承認を取得 (P-10、 constitution 改訂の場合)。
  3. ADR を Accepted status で本 cluster に作成。
  4. constitution / rules / folio-self-spec を編集し、 SemVer bump + delta marker を inline 追加。
  5. changes package を archive/ へ移動。

例外: folio 初の ADR (ADR-0001) は changes/ dir 未存在の bootstrap 段階で起票されたため、 proposal step を skip した折衷を採用。 後続 ADR は本手順を遵守する MUST。
